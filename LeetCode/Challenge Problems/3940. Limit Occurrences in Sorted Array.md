# 3940. Limit Occurrences in Sorted Array

## Problem Description
You are given a **sorted** integer array `nums` and an integer `k`.

Return an array such that each **distinct** element appears **at most** `k` times, while preserving the relative order of the elements in `nums`.

### Example 1
- **Input:** `nums = [1,1,1,2,2,3]`, `k = 2`
- **Output:** `[1,1,2,2,3]`
- **Explanation:**
  Each element can appear at most 2 times.
  - The element `1` appears 3 times, so only 2 occurrences are kept.
  - The element `2` appears 2 times, so both occurrences are kept.
  - The element `3` appears 1 time, so it is kept.
  Thus, the resulting array is `[1, 1, 2, 2, 3]`.

### Example 2
- **Input:** `nums = [1,2,3]`, `k = 1`
- **Output:** `[1,2,3]`
- **Explanation:**
  All elements are distinct and already appear at most once, so the array remains unchanged.

---

## Approach 1: Auxiliary Array (Keeping Counts)

### Intuition
Since the input array is already sorted, identical elements are contiguous. We can iterate through `nums` and keep track of the count of the current element. If the count is less than or equal to `k`, we append it to a helper list `stor_arr` and finally modify `nums` in-place.

### Implementation
```python
class Solution:
    def limitOccurrences(self, nums: list[int], k: int) -> list[int]:
        stor_arr=[]
        count=0
        for i in range(len(nums)):
            if i==0 or nums[i]!=nums[i-1]:
                count=1
            else:
                count+=1
                
            if(count<=k):
                stor_arr.append(nums[i])

        nums[:]=stor_arr  
        return nums 
```

### Complexity Analysis
- **Time Complexity:** $\mathcal{O}(N)$ where $N$ is the length of `nums`.
- **Space Complexity:** $\mathcal{O}(N)$ because of the extra `stor_arr` auxiliary array.

## Approach 2: Two-Pointer (In-Place)

### Intuition
To avoid using extra memory, we can use a two-pointer approach (`l` and `r`).
- The pointer `r` scans through the array to find identical elements and count their frequency.
- The pointer `l` serves as the write index for elements to be kept.
- For each unique element, we write it at most `min(k, count)` times at the index `l`, then advance `l`.
- Finally, we slice `nums` to size `l` in-place.

### Implementation
```python
class Solution:
    def limitOccurrences(self, nums: list[int], k: int) -> list[int]:
        l,r=0,0
        while r<len(nums):
            count=1
            while r+1<len(nums) and nums[r]==nums[r+1]:
                count+=1
                r+=1
            for i in range(min(k,count)):
                nums[l]=nums[r]
                l+=1
            r+=1
        nums[:] = nums[:l]
        return nums    
```

### Complexity Analysis
- **Time Complexity:** $\mathcal{O}(N)$ where $N$ is the length of `nums`. Each element is visited at most twice (once by the inner loop, once by the outer).
- **Space Complexity:** $\mathcal{O}(1)$ auxiliary space because we modify the array in-place.

## 💡 Key Takeaway / Pro-Tip
> [!TIP]
> This trick completely removes the need for extra memory. Whenever you encounter a LeetCode problem with a sorted array that mentions "in-place" or "$O(1)$ space", this two-pointer lookback method is almost always the hidden key!
